Disk Crash

Aug 21st, 2012

Ouch! My Disk Crashed

My MacBook Pro became unusable and required a complete repartition and several reformats before I could resuscitate it. Recovering from the disk crash was surprisingly easy, there were a few hitches. Being that, without a duplicate computer, the only time that you can see how well your backup strategy works and correct it, I’ve been using this disk crash as an opportunity to bolster my backup strategy.

While I had some gaps in my backup strategy, within 2 hours of my disk crash I had my backup computer running, complete with my data files, shell scripts, config files, etc…

File Distribution

In my mind, file backup and file distribution across multiple machines are intimately related. File backup is, well it’s file backup. I use file distribution to keep various files synchronized across my devices. It should be noted that while the goal of file distribution is synchronization of data across computers, distributing files to other computers effectively backs them up.

My primary computers are my MacBook Pro and an HP laptop running Ubuntu Linux. My mobile is a Samsung Infuse running CyanogenMod 10, a customized version of Android 4.1 Jelly Bean. I also rent a VPS cloud computer running Ubuntu. Lastly, I have an Asus running Windows 7. I primarily use te Windows computer for streaming media (I don’t dig Windows).

I use file distribution to achieve two goals. I like to keep my configuration files consistent across my MacBook and two Ubuntu computers. This includes my .bashrc and my .emacs site startup files. While I want those files to stay in sync to make the management of them simpler, there’s generally no rush in synchronizing them.

Another example of my file distribution involves my PIM. I use Org Mode as my PIM and want the data files to be synchronized across my MacBook, two Linux boxes, and my Android. I want those files synchronized immediately as I frequently move from computer to computer and always want my PIM data to be up to date.

Prioritization Of File Backup And File Distribution

My backup and distribution strategy categorizes files using two metrics:

File Importance
This describes how critical each file is. Files in ~/Downloads are typically not very important while the text for this blog entry is very important (at least to me).
Synchronization Delay
As I move between multiple computers, my productivity is related to how frequently some files are synchronized across the computers. Some I want optimized as soon as possible while other files can take hours or even days to get synchronized without affecting my productivity. Some low priority files, such as those in ~/Downloads don’t ever need to be synchronized across machines.

In practice this leads to four types of files:

High/Timely
Files that are very important and need to be synchronized in a timely fashion. My PIM files fall into this category.
High/Delayed
Important files that don’t need to timely synchronization. Configuration files such as my .bashrc are High/Delayed.
Low/Delayed
Files that are convenient to have across my machines but aren’t essential to my immediate productivity. For example, I frequently download manuals for various appliances that I own. I like to have them handy but it’s fine if they aren’t synched immediately. As I can easily download them again, they’re of relatively low importance.
Everything else
These are files that are of significantly low importance that I don’t worry too much about them.

My backup and distribution strategy varies by which quadrant (I’m excluding the N/A boxes) a file is in.

Time Tracking, Part 2: Org Mode File Format

Aug 5th, 2012

A Brief Introduction To Org Mode File Syntax

I’ve decided that my time tracking solution requires some basic understanding of the file format of Org Mode as Org Mode syntax pervades the system. I’ve done my best to cover the smallest amount of Org Mode syntax that is needed to understand the examples I’ll of my time tracking system.

It is frequently said that when you buy a house the three most important things to look for are location, location, and location. When it comes to Org Mode, the three most important things to remember are Org Mode file format is plain text, Org Mode file format is plain text, and Org Mode file format is plain text.

By plain text I mean:

  • You don’t need a GUI or any other program to read an Org Mode file.
  • Any text editor can be used to create Org Mode files.
  • Org Mode files are highly compatible with standard text file utilities such as grep(1).

While Org Mode’s use of plain text has many advantages, the files tend to look a bit intimidating to a newcomer.

While this posting only discusses Org Mode’s file format, I use Org Mode for its incredibly rich organizational functionality.

Hierarchical Outlines

Org Mode encourages you to store your data in an outline. A level one headline is simply a line of text that begins with a single asterisk (*). A level two headline begins with two asterisks (**), etc… 

* Work
** Work hard.
Finish my project ahead of schedule.
** Ask boss for raise.
Remember to compliment him lots when asking.

* Personal
** Clean up after my kids.
Grumble while cleaning up.
** Tell my kids that they should clean up after themselves.
Laugh quietly to myself as I know it won't help.
